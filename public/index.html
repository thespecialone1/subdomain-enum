Discovery Methods

Understanding each subdomain discovery technique

Wayback Machine

Searches Internet Archive's historical web crawl data to discover subdomains from archived URLs. Excellent for finding old or deprecated subdomains that may still be active.

Timeout: 5 minutes | Source: web.archive.org

Certificate Transparency

Queries public SSL/TLS certificate logs to find subdomains from certificate Subject Alternative Names. Great for discovering active HTTPS subdomains.

Timeout: 5 minutes | Source: crt.sh

DNS Brute Force

Tests 500+ common subdomain patterns using multiple DNS servers with concurrent resolution. Most reliable method for finding active subdomains.

Timeout: 10 minutes | Concurrency: 50 queries

Search Engine Scraping

Automated Google search with "site:" operator to extract subdomains from search results. Respects search engine rate limits.

Timeout: 5 minutes | Note: Rate limited

Best Practices

Tips for effective subdomain enumeration

Target Selection

  • Only scan domains you own or have explicit permission to test
  • Start with passive methods (Wayback, Certificate Transparency) first
  • Use active methods (DNS, Search) for comprehensive discovery
  • Consider rate limits and respect robots.txt

Performance Optimization

  • Adjust DNS concurrency based on your network capacity
  • Use shorter timeouts for faster scanning
  • Enable HTTPS-first probing for modern domains
  • Monitor resource usage during large scans

Advanced Usage

  • Combine multiple sources for comprehensive coverage
  • Export results for integration with other security tools
  • Use custom wordlists for target-specific patterns
  • Set up API access for automated workflows

Troubleshooting

Common issues and solutions

No Results Found

  • Verify domain name is spelled correctly
  • Check internet connectivity
  • Try different source combinations
  • Some APIs may be temporarily unavailable

Slow Performance

  • Reduce DNS concurrency settings
  • Increase timeout values for unstable connections
  • Disable resource-intensive sources temporarily
  • Check system resource usage

Connection Errors

  • Check firewall settings for outbound connections
  • Verify DNS server accessibility
  • Try different DNS servers if queries fail
  • Ensure port 8080 is available for the web interface
